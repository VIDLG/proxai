# ProxAI 协议模型总览

ProxAI 把**数据处在代理链路的哪个阶段**和**这个阶段使用哪种 wire protocol**拆成两个独立概念。

<PipelineStrip
  steps={[
    { title: 'inbound_request', description: '客户端发给 ProxAI 的原始请求。' },
    { title: 'provider_request', description: 'ProxAI 准备发给选中上游 provider 的请求。' },
    { title: 'upstream_response', description: '上游 provider 返回给 ProxAI 的响应。' },
    { title: 'outbound_response', description: 'ProxAI 返回给客户端协议的响应。' },
  ]}
/>

## 协议轴

| 协议值 | 含义 |
|---|---|
| `openai_responses` | OpenAI Responses API |
| `openai_chat_completions` | OpenAI Chat Completions API |
| `anthropic_messages` | Anthropic Messages API |

<ProxaiCallout type="note" title="路由不是翻译">
  route 可以选择一个协议不同于入站请求的 provider。这时必须存在显式转换路径；未支持的协议对会明确失败。
</ProxaiCallout>
