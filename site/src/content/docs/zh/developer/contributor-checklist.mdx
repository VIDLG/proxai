# 贡献者检查清单

修改 ProxAI 行为并准备提交 PR 前，使用这份清单检查。它刻意偏操作化：同步代码路径、测试和文档，让项目保持一致。

<ProxaiCallout type="tip" title="小代理，显式变更">
  优先做聚焦改动，保持 ProxAI 作为本地兼容代理的定位。除非有明确产品决策，不要把 routing 或 provider 支持扩展成泛 AI gateway。
</ProxaiCallout>

## 隐私检查

- 不要提交 `config.toml`、API keys、private captures、logs 或包含 prompt 的本地 fixtures。
- 脱敏 fixtures 应保持最小，不包含 private upstream URLs 或 credentials。
- 日志不应包含 request bodies、Authorization headers、API keys 或 private prompts。
