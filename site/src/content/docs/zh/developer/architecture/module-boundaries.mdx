# 模块边界

本页是维护者判断代码归属的地图。拿不准时，保持协议转换显式，并避免把 provider-specific 行为混入通用 translation。

<ProxaiCallout type="caution" title="不要混淆协议和 provider 边界">
  Provider 名称是用户标签。协议名称描述 wire 行为。Provider 模块可以适配本地 provider 怪癖，但通用协议到协议的 shape 改写应放在 `src/translation/`。
</ProxaiCallout>
