# 配置流程

本页说明运行时配置如何从文件和 CLI overrides 进入已校验的运行时状态。

<ProxaiCallout type="note" title="配置是运行时事实来源">
  `config.toml` 是本地运行时 source of truth。`config.example.toml` 是仓库追踪的示例来源。文档跟随代码和已验证默认值，而不是愿景式配置示例。
</ProxaiCallout>

## 新增设置时

1. 在 `src/config.rs` 添加 typed field。
2. 在 `config.example.toml` 添加简洁示例/注释。
3. 接入 `src/main.rs`、`src/lib.rs` 或归属模块。
4. 如果行为面向用户，更新用户文档。
5. 更新 reference 文档和默认值/校验测试。
