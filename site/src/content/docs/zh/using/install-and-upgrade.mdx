# 安装与升级

当你需要处理运维路径时使用本页：获取二进制、在本地运行、定位配置，并在不丢失本地状态的前提下升级。

<ProxaiCallout type="note" title="运行时配置保持本地">
  `config.toml` 有意保持本地并被 git 忽略。仓库中的 `config.example.toml` 是参考示例；真正的运行时配置位于你的应用目录。
</ProxaiCallout>

## 应用目录

ProxAI 会把本地运行状态放在应用目录。具体路径取决于平台和启动配置。

## 本地源码构建

如果从源码构建，在替换工作中的二进制之前，优先运行仓库验证命令：

```sh
just check
```

只改站点时使用：

```sh
just site check
```

## 安全回滚

通常只需要把二进制替换回旧版本，同时保留同一个本地 `config.toml`。如果新版本引入了配置字段，只移除旧版本无法解析的字段。

<ProxaiCallout type="caution" title="不要把捕获产物作为回滚材料提交">
  captures 和 logs 可能包含 prompt、工具参数、输出、provider payload 和上游标识。除非经过有意脱敏，否则保持本地。
</ProxaiCallout>
