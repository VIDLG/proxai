# 客户端接入

当本地客户端或 SDK 需要填写 base URL、API key、model、endpoint 或 streaming 设置时，看本页。

<ProxaiCallout type="note" title="本地客户端 key 不是 provider key">
  Provider 认证由 `config.toml` 控制。如果客户端 UI 要求填写 API key，可以填一个本地占位值；真正的上游 key 应放在选中的 provider 配置里。
</ProxaiCallout>

## 模型名与路由

客户端发送的模型会先被匹配，然后才做 provider model rewrite：

```toml
[[routing.routes]]
request_protocol = "openai_chat_completions"
match_kind = "exact"
model_pattern = "MiniMax-M3-preview"
provider = "minimax"
upstream_model = "MiniMax-M3"
```

- `model_pattern` 匹配客户端发送的值。
- `provider` 选择上游配置。
- `upstream_model` 是 ProxAI 发给 provider 的模型名。

详见[路由与 Provider](/zh/using/routing-and-providers) 和[路由匹配](/zh/reference/route-matching)。

## 错误格式

交互式调试优先使用可读 text；SDK 期望结构化错误时使用 JSON：

```toml
[error_responses]
format = "json"
```

见[错误响应](/zh/reference/error-responses)。
