# Zed 接入

当你希望 Zed 访问 ProxAI，而不是直接访问 provider 时，用这页。ProxAI 保持本地运行，接收 OpenAI 兼容请求，再转发到配置好的上游 provider。

<ProxaiCallout type="note" title="本地代理边界">
  Zed 把请求发到 ProxAI 的本地 base URL。ProxAI 再根据入站协议、模型和 provider 配置路由。长期路由行为应放在 `config.toml`，不要散落在临时客户端设置里。
</ProxaiCallout>

## 隐私提示

Zed 接入可能包含私有 prompt、工具调用、文件片段和 provider 输出。ProxAI 不应记录 request bodies、`Authorization` headers、API keys 或不必要的私有上游 URL 细节。Capture 是本地调试产物，应视为敏感数据。

分享日志或 capture 前，先看 [隐私与本地数据](/zh/using/privacy-and-local-data) 和 [环境与文件](/zh/reference/environment-and-files)。
