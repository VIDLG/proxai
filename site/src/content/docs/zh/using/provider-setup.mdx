# Provider 设置

Provider 是一个命名上游配置。**Provider 名称**是你的标签；**provider protocol** 控制出站 wire 行为。

<ProxaiCallout type="note" title="名称只是标签">
  `providers.openai`、`providers.minimax` 或 `providers.work` 都是任意名称。只有 `providers.<name>.protocol` 决定 ProxAI 向上游发送 OpenAI Responses、Chat Completions 还是 Anthropic Messages。
</ProxaiCallout>

## 最小 provider

```toml
[providers.openai]
protocol = "openai_responses"
base_url = "https://api.openai.com"
api_key = "..."
```

## 作为默认 provider

```toml
[routing.default_provider_names]
openai_responses = "openai"
openai_chat_completions = "openai"
```

只有没有显式 route 匹配时才使用 defaults。见[行为契约](/zh/reference/behavior-contracts#路由与转换)。

## 路由某个模型

```toml
[[routing.routes]]
request_protocol = "openai_responses"
match_kind = "exact"
model_pattern = "my-model"
provider = "anthropic_upstream"
upstream_model = "claude-like-model"
```

当同一个 model pattern 需要按 endpoint 表现不同时，使用 `request_protocol`。

## 认证行为

<ReferenceTable
  columns={['Provider 协议', '上游认证 header']}

客户端传入的 `Authorization` 不作为上游 provider 认证。见[客户端接入](/zh/using/client-integration)。
