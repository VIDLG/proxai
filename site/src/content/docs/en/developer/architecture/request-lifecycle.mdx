# Request Lifecycle

This page describes the source-level request path. User-facing behavior lives in [Using ProxAI](/en/using) and protocol behavior lives in [Protocol Guide](/en/protocol).

<ProxaiCallout type="note" title="Follow the data phase">
  The easiest way to reason about the code is to track which phase owns the data: `inbound_request`, `provider_request`, `upstream_response`, or `outbound_response`.
</ProxaiCallout>
