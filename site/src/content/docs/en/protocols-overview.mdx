# Protocol Overview

ProxAI separates **where data is in the proxy pipeline** from **which wire protocol that data uses**.

<PipelineStrip
  steps={[
    { title: 'inbound_request', description: 'The original client request received by ProxAI.' },

    { title: 'upstream_response', description: 'The response returned by the upstream provider.' },
    { title: 'outbound_response', description: 'The response ProxAI returns to the client-facing protocol.' },
  ]}
/>

## Protocol axis

| Protocol | Meaning |
|---|---|
| `openai_responses` | OpenAI Responses API |
| `openai_chat_completions` | OpenAI Chat Completions API |
| `anthropic_messages` | Anthropic Messages API |

<ProxaiCallout type="note" title="Routing is not translation">
  A route may select a provider whose protocol differs from the inbound request protocol. That requires an explicit translation pair; unsupported pairs fail explicitly.
</ProxaiCallout>
