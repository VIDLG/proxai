# Client Integration

Use this page when a local client or SDK asks for a base URL, API key, model, endpoint, or streaming setting.

<ProxaiCallout type="note" title="The local client key is not the provider key">
  Provider authentication is owned by `config.toml`. If a client UI requires an API key field, use a dummy local value there and keep the real upstream key in the selected provider config.
</ProxaiCallout>

## Model names and routes

The client-facing model is matched before provider model rewrite:

```toml
[[routing.routes]]
request_protocol = "openai_chat_completions"
match_kind = "exact"
model_pattern = "MiniMax-M3-preview"
provider = "minimax"
upstream_model = "MiniMax-M3"
```

- `model_pattern` matches what the client sends.
- `provider` selects the upstream configuration.
- `upstream_model` is what ProxAI sends to the provider.

See [Routing and Providers](/en/using/routing-and-providers) and [Route Matching](/en/reference/route-matching).

## Error format

Use readable text for interactive debugging, and JSON when an SDK expects structured errors:

```toml
[error_responses]
format = "json"
```

See [Error Responses](/en/reference/error-responses).
