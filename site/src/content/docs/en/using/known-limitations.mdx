# Known Limitations

This page sets expectations for what ProxAI intentionally does not do today. It is a user-facing summary; exact protocol support lives in [Compatibility Matrix](/en/reference/compatibility-matrix), and exact configuration fields live in [Configuration Reference](/en/reference/configuration).

<ProxaiCallout type="note" title="Limitations are design boundaries">
  ProxAI is a small local compatibility proxy. It should not grow into a generic AI gateway unless the project explicitly chooses that direction.
</ProxaiCallout>

## If you hit a limit

1. Confirm the behavior in the relevant reference page.
2. Reproduce with the smallest request and the narrowest useful capture phase.
3. Decide whether the change belongs to provider compatibility, protocol translation, routing, or docs.
4. If it changes user-visible behavior, update both user docs and reference docs.

For local debugging paths, see [Debugging Workflow](/en/using/debugging-workflow) and [Troubleshooting](/en/using/troubleshooting).
