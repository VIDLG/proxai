# Provider Setup

A provider is a named upstream configuration. The **provider name** is your label; the **provider protocol** controls outbound wire behavior.

<ProxaiCallout type="note" title="Names are labels">
  `providers.openai`, `providers.minimax`, or `providers.work` are arbitrary names. Only `providers.<name>.protocol` tells ProxAI whether to send OpenAI Responses, Chat Completions, or Anthropic Messages upstream.
</ProxaiCallout>

## Minimal provider

```toml
[providers.openai]
protocol = "openai_responses"
base_url = "https://api.openai.com"
api_key = "..."
```

## Use as default

```toml
[routing.default_provider_names]
openai_responses = "openai"
openai_chat_completions = "openai"
```

Defaults are used only when no explicit route matches. See [Behavior Contracts](/en/reference/behavior-contracts#routing-and-conversion).

## Route one model

```toml
[[routing.routes]]
request_protocol = "openai_responses"
match_kind = "exact"
model_pattern = "my-model"
provider = "anthropic_upstream"
upstream_model = "claude-like-model"
```

Use `request_protocol` when the same model pattern should behave differently by endpoint.
